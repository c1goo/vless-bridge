# VLESS Bridge

VLESS-прокси **только для браузера**. Вставляете **ссылку-подписку** (subscription URL) или сырую `vless://` ссылку — IP меняется в Chrome, вся остальная система (Telegram, игры, другие приложения) продолжает ходить напрямую.

Архитектура — как у Happ/v2rayN, но спрятанная за расширением:

```
Chrome ──chrome.proxy──▶ SOCKS5 127.0.0.1:<порт> ──▶ xray-core ──VLESS──▶ ваш сервер
   ▲                                                    ▲
   └── расширение ──Native Messaging──▶ helper (Go) ────┘
```

- **extension/** — расширение Chrome (Manifest V3): UI, список серверов, вкл/выкл, управление прокси.
- **helper/** — нативный компонент на Go: принимает команды от расширения, генерирует конфиг xray и запускает/останавливает xray-core.

> **Открытый код.** Весь код расширения и helper'а лежит в этом репозитории — можно прочитать и собрать самому. Готовые бинарники (`helper`, `xray`, geo-базы) в git **не** хранятся: их берут со страницы **Releases** этого репозитория либо собирают по инструкции ниже. Без helper'а расширение работать не будет — Chrome сам не умеет ходить в VLESS, всю сетевую работу делает helper + xray.

**Подписки.** Ссылка вида `https://provider.com/sub/<id>`. Расширение скачивает её и определяет формат автоматически:

| Формат подписки | Как обрабатывается |
|---|---|
| **Готовый конфиг xray** (JSON с `outbounds`; один объект или массив) | Конфиг используется как есть — вместе с routing-правилами и DNS провайдера. Заменяется только блок `inbounds`: вместо зашитого порта (обычно 10808) подставляется свободный локальный SOCKS-порт. Работает с любым протоколом, который умеет xray: VLESS, VMess, Trojan, Shadowsocks. |
| **Список ссылок** (base64, base64 URL-safe или plain text построчно) | Разбираются `vless://` ссылки. Другие схемы (`vmess://`, `trojan://`, `ss://`) пока пропускаются с уведомлением. |

Кнопка ↻ обновляет список, если провайдер поменял серверы.

Поддерживаемые параметры ссылок: транспорт `tcp` / `ws` / `grpc`, шифрование `tls` / `reality` / `none`, включая `flow`, `sni`, `fp`, `pbk`, `sid`, `alpn`, `path`, `host`.

## Сборка helper'а

Нужен Go 1.21+ (https://go.dev/dl/). Из папки `helper/`:

```bash
# macOS (Apple Silicon)
GOOS=darwin GOARCH=arm64 go build -o install/helper .

# macOS (Intel)
GOOS=darwin GOARCH=amd64 go build -o install/helper .

# Windows
GOOS=windows GOARCH=amd64 go build -ldflags -H=windowsgui -o install/helper.exe .
```

Зависимостей нет — только стандартная библиотека, собирается из любой ОС под любую.

## xray-core

Скачайте официальный релиз xray-core: https://github.com/XTLS/Xray-core/releases

- macOS: `Xray-macos-arm64-v8a.zip` (или `64` для Intel) → положите бинарник `xray` в `helper/install/`
- Windows: `Xray-windows-64.zip` → положите `xray.exe` в `helper/install/`

Из того же архива положите рядом (`helper/install/`) файлы **`geoip.dat`** и
**`geosite.dat`** — они нужны xray для routing-правил вида `geosite:category-ru` /
`geoip:ru`. Многие подписки их используют, и без этих файлов конфиг не загрузится
(`failed to load geosite … no such file or directory`).

## Установка

**1. Расширение.** `chrome://extensions` → включите «Режим разработчика» → «Загрузить распакованное» → выберите папку `extension/`. Скопируйте **ID расширения** (строка вида `abcdefgh…`).

**2. Helper.**

macOS:
```bash
cd helper/install
chmod +x install-macos.sh
./install-macos.sh <ID_расширения>
```

Windows (PowerShell):
```powershell
cd helper\install
powershell -ExecutionPolicy Bypass -File install-windows.ps1 -ExtensionId <ID_расширения>
```

**3. Перезапустите Chrome.**

## Использование

1. Откройте попап расширения, вставьте ссылку-подписку (или сырую `vless://`), нажмите «+».
   Из подписки подтянутся все VLESS-серверы сразу.
2. Выберите сервер в списке → «Подключить».
3. Кнопка «IP?» покажет текущий внешний IP браузера — удобно проверить, что туннель работает.

При отключении (или падении helper'а) расширение автоматически снимает прокси, чтобы браузер не остался без сети.

## Известные ограничения (v0.1)

- Требуется Chrome 116+ (более старые версии выгружают service worker и рвут соединение с helper'ом).
- Нет замера пинга/скорости серверов (план на следующую версию).
- Подписка обновляется только вручную (кнопка ↻), автообновления по расписанию пока нет.
- Chrome не даёт расширениям подменять `User-Agent` при запросе подписки. Если панель провайдера отдаёт конфиг только «известным» клиентам (v2rayNG, Streisand и т.п.) и отвечает 403 — потребуется добавить `declarativeNetRequest` для подмены заголовка.
- Расширение запрашивает `host_permissions` на все домены — это нужно, чтобы скачивать подписку с произвольного адреса. Перед публикацией в Store стоит перейти на `optional_host_permissions` и запрашивать доступ только к домену конкретной подписки.
- Нет split-tunneling по сайтам (можно добавить через PAC-скрипт).
- WebRTC может раскрывать реальный IP — стоит добавить управление `privacy.network.webRTCIPHandlingPolicy` (permission `privacy`) в следующей версии.
- Установка helper'а пока ручная (скрипт). Для продукта: собрать .pkg/.msi установщик, который делает всё сам.
